# 流程定义处理引擎 - 需求文档

## 一、项目概述

设计一套通用的流程定义处理引擎，支持可视化配置、节点事件处理、业务分离、回退节点、多人并行审核等功能。

## 二、核心需求

### 2.1 流程节点可配置

- 支持可视化拖拽式流程设计
- 节点类型：开始、审批、条件网关、并行网关、结束
- 节点属性：名称、标识、位置、审核人配置

### 2.2 节点事件处理和业务分离

- 使用策略模式实现事件处理器接口
- 支持节点进入事件、审批前事件、审批后事件、跳过事件
- 业务方实现接口即可，无需修改引擎核心代码

### 2.3 支持回退节点配置

- 支持指定回退：可配置节点可回退到哪些节点
- 流程设计时可配置节点是否允许回退

### 2.4 支持多人并行审核

- 会签：所有人通过才算通过
- 或签：任意一人通过即可
- 支持按部门、角色、用户等配置审核人

## 三、候选人类型

| 类型 | 说明 |
|------|------|
| USER | 指定用户 |
| ROLE | 指定角色 |
| DEPARTMENT | 指定部门 |
| DEPARTMENT_LEADER | 部门负责人 |
| INITIATOR_DEPT | 发起人所属部门 |
| DEPT_UPWARD | 发起人部门及上级部门 |
| EXPRESSION | 表达式计算 |

## 四、审批规则

| 类型 | 说明 |
|------|------|
| SEQUENCE | 顺序审批 |
| COUNTER_SIGN | 会签（全员通过） |
| OR_SIGN | 或签（任一通过） |

## 五、流程状态

| 状态 | 说明 |
|------|------|
| RUNNING | 进行中 |
| COMPLETED | 已完成 |
| TERMINATED | 已终止 |
| ROLLED_BACK | 已回退 |

## 六、任务状态

| 状态 | 说明 |
|------|------|
| PENDING | 待办 |
| COMPLETED | 已办 |
| REJECTED | 已拒绝 |
| DELEGATED | 已转办 |

## 七、扩展性设计

1. 节点类型可扩展：新增枚举值即可
2. 事件处理器可扩展：实现接口即插即用
3. 候选人策略可扩展：实现CandidateStrategy接口
4. 条件表达式可扩展：支持SpEL脚本引擎
